GP Settings

Allowed Symbols
Choose which mathematical operations the GP algorithm can use to build expressions. More symbols = more expressive but slower search. constant and variable are always required. Select function primitives
Primary
Number of evolutionary generations. Higher = more time to evolve better solutions but slower. Lower = faster but may find simpler/worse fits. Typical range: 10–100. generations
Number of candidate solutions in each generation. Higher = more diversity, better exploration, but more memory and time. Lower = faster iterations but risk premature convergence. Typical: 100–1000. population_size
Maximum tree length (number of nodes) for candidate expressions. Higher = allows more complex expressions. Lower = forces simpler, more interpretable models. Typical: 10–50. tree_length
Maximum tree depth (height). Higher = deeper nesting of operations. Lower = shallower, often simpler models. Typical: 5–30. tree_depth
Secondary
Probability of mutating an individual each generation (0–1). Higher = more exploration, can escape local optima. Lower = more exploitation of good solutions. Typical: 0.05–0.3. mutation_rate
Number of individuals competing in tournament selection. Higher = stronger selection pressure, favors best individuals. Lower = weaker pressure, more diversity. Typical: 2–7. tournament_size
Number of best individuals preserved unchanged to the next generation. Higher = more exploitation. Lower = more exploration. Typical: 1–5. elites
Number of iterations for optimizing constants in expressions. Higher = better-tuned coefficients but slower. Lower = faster but coefficients may be rough. Typical: 1–20. Set 0 to disable. parameter_optimization_iterations
Advanced
Enable Keijzer linear scaling (offset + scale * expression). Improves fit quality by automatically adjusting constants. Generally keep enabled. use_linear_scaling
Seed for the random number generator. Set a number ≥ 0 for reproducible results. Set to -1 for random initialization each run. seed
Visualization
Delay between generation updates (milliseconds). Higher = more time to observe each generation's population. 0 = as fast as possible. Useful for investigating evolution step by step. step_delay_ms
0 ms
Maximum number of population curves to display per generation. Higher = denser visualization of population diversity. Lower = less visual clutter and faster rendering. Curves are randomly sampled. max_displayed_curves
